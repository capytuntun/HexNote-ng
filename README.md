# 報告筆記系統

多人 Markdown 報告 / 筆記工具。每個帳號只看得到自己的筆記，並可把單篇筆記分享給其他帳號（唯讀或可編輯）。

> [!IMPORTANT]
> **這套系統從 v2 起改為前後端架構。** 舊版是純前端、資料存在瀏覽器 IndexedDB；現在資料存在伺服器的 `server/data/data.db`，**必須啟動伺服器才能使用，也不再能用 `file://` 直接開啟**。
> 首次登入時，若偵測到這台瀏覽器還留有舊版的本機筆記，畫面下方會出現提示，可一鍵上傳到你的帳號（舊資料不會被刪除）。

## 啟動

需要 **Node.js 22 以上**（用到內建的 `node:sqlite`）。**不需要 npm install**——整套後端只用 Node 內建模組。

```bash
node server/server.js
```

預設在 <http://localhost:8080>。首次啟動會印出邀請碼。

### 環境變數

| 變數 | 預設 | 說明 |
| --- | --- | --- |
| `PORT` | `8080` | 監聽埠 |
| `REGISTER_MODE` | `invite` | `open`／`invite`／`closed` |
| `ADMIN_USERNAME` | `admin` | 管理員帳號名稱 |
| `ADMIN_PASSWORD` | 隨機 | 未設定則首次啟動隨機產生並印出，且**強制首次登入時更改** |
| `INVITE_CODE` | 隨機 | 未設定則每次啟動隨機產生，**請自行設定固定值** |
| `TRUST_PROXY` | `0` | 放在 nginx／Caddy 後方時設 `1`，才會正確判斷 HTTPS 與來源 IP |
| `REQUIRE_HTTPS` | `1` | Cookie 加 `Secure` 旗標並啟用 HSTS。**僅 localhost 測試時才設 0** |
| `DATA_DIR` | `server/data` | 資料庫位置 |
| `SESSION_TTL_DAYS` | `14` | 登入有效天數 |
| `BUSY_TIMEOUT_MS` | `5000` | 資料庫被鎖住時最多等待多久（0 = 立刻失敗） |
| `MIN_PASSWORD_LENGTH` | `12` | 最短密碼長度 |

備份：`node server/backup.js [輸出檔]` — 用 `VACUUM INTO` 產生單一完整檔案，可在伺服器運行中執行。

### 搬到其他主機

整包是可攜的：沒有寫死的路徑、沒有 `node_modules`、沒有平台專屬程式碼。唯一的硬性需求是**目標主機要有 Node 22 以上**（後端用到內建的 `node:sqlite`）。

**只搬程式碼**（重新開始）：把整個資料夾複製過去，`node server/server.js` 即可。會自動建立新的資料庫與管理員帳號。

**連同資料一起搬**：

```bash
# 1. 舊主機：備份成單一檔案（可在伺服器運行中執行）
node server/backup.js move.db

# 2. 新主機：解壓程式碼後放進去
mkdir -p server/data && cp move.db server/data/data.db
node server/server.js
```

> [!CAUTION]
> **不要直接複製 `server/data/` 資料夾。** WAL 模式下，最近的寫入全都在 `data.db-wal` 裡——實測時 `data.db` 只有 4 KB，而 848 KB 的資料都在 WAL。**只複製 `data.db` 開起來會是「no such table: users」，資料全失。**
> 要嘛用 `server/backup.js`（推薦，單一檔案且不需停機），要嘛先停掉伺服器讓它把 WAL 寫回，再把 `data.db`、`data.db-wal`、`data.db-shm` **三個一起**複製。

**不要複製的東西**：`server/data/server.lock`（會被自動忽略，因為它記錄了來源主機名稱）、`*.log`。`.gitignore` 已經排除 `server/data/`。

### 部署在公網（重要）

這套系統裝的是滲透測試報告——憑證、雜湊、客戶資料。放上公網前請務必：

1. **一定要有 HTTPS。** 本程式不自己處理 TLS，請放在反向代理後方並設 `TRUST_PROXY=1`：
   ```
   # Caddy 一行搞定（自動申請憑證）
   your.domain.com { reverse_proxy localhost:8080 }
   ```
   沒有 HTTPS 的話，帳號密碼與 session cookie 都是明文傳輸。
2. **固定邀請碼並考慮用完關閉**：`REGISTER_MODE=closed`。
3. **備份 `server/data/data.db`**（含 WAL 檔）。這個檔案就是全部資料。
4. **它的安全性沒有經過第三方稽核。** 我實作了下列防護並逐項測過，但這不等於「保證安全」；資料愈敏感，愈該考慮只放在 VPN／內網後方。

### 資料庫與並行

資料存在單一的 `server/data/data.db`（SQLite，WAL 模式）。針對「資料庫被鎖住」做了這些處理：

- **`busy_timeout = 5000`** — SQLite 預設是 `0`，也就是一碰到鎖就**立刻失敗**。這會讓任何短暫的競爭（備份、另一個行程、順手開的 sqlite shell）直接變成存檔失敗。現在會等待。
- **WAL 模式** — 寫入進行中讀取不受影響，所以有人在存檔時別人仍可正常瀏覽。
- **`BEGIN IMMEDIATE` 交易** — 多步驟的寫入（停用帳號＋踢掉 session、刪除帳號）要嘛全成功要嘛全不做。用一般的 `BEGIN` 會先當讀者再升級成寫者，那正是 SQLite 典型的死鎖來源。
- **自動重試** — `busy_timeout` 涵蓋不了 WAL 的 `SQLITE_BUSY_SNAPSHOT`（兩個寫者從不同快照出發），這種要重跑整個操作才有用。
- **忙碌時回 503 而非 500** — 前端收到 503 會自動重試兩次，所以短暫鎖定對使用者是無感的。
- **單一實例保護** — 同一個 `DATA_DIR` 只允許一個伺服器（`server.lock`）。**兩個伺服器寫同一個 data.db 是最常見的鎖定來源**，而且換個埠並不會避開它。行程若異常結束，殘留的 lock 會在下次啟動時自動清除。
- **優雅關閉** — `Ctrl+C` 會把 WAL 寫回 `data.db` 再退出，避免資料只留在 `-wal` 檔裡。
- **`synchronous = NORMAL`** — WAL 模式的建議搭配：不會造成資料庫損毀，僅在**主機斷電**時可能損失最後幾筆交易。

> [!TIP]
> **不要把 `server/data/` 放進雲端同步資料夾**（OneDrive、Dropbox、Google Drive）或網路磁碟。同步工具會在 SQLite 寫入時抓住檔案，這是 SQLite 損毀最經典的成因。要備份請用 `sqlite3 data.db ".backup out.db"`，或在伺服器停止時複製。

### 已實作的安全措施

- 密碼用 **scrypt** ＋ 每人隨機 salt，比對用 constant-time；登入失敗訊息不透露帳號是否存在
- Session token 只在資料庫存 **SHA-256 雜湊**（DB 外洩也無法直接冒用），Cookie 為 `HttpOnly` ＋ `SameSite=Strict`（＋ `Secure`）
- 登入**限速鎖定**（同帳號＋IP）
- **每一支 API 都在伺服器端檢查權限**，前端傳來的擁有者資訊一律不採信
- 圖片授權綁定「你讀得到某篇引用它的筆記」，避免猜 id 撈別人的截圖（IDOR）
- 靜態檔案防路徑穿越；`server/` 目錄（含資料庫）不對外提供
- CSP（`script-src 'self'`，無 inline script）、`nosniff`、`frame-ancestors 'none'`、HSTS
- 所有 SQL 皆為 prepared statement

## 管理員

首次啟動會**自動建立管理員帳號**，並把密碼印在終端機上：

```
  ┌─ 已建立管理員帳號 ─────────────────────────
  │  帳號: admin
  │  密碼: qF0MnCofH0Y8hCha
  │
  │  ⚠ 這是隨機產生的密碼，只會顯示這一次。
  │    首次登入時會要求你立刻更改。
  └────────────────────────────────────────────
```

> [!CAUTION]
> **預設密碼是隨機產生的，不是固定值**，而且首次登入會**強制更改**。
> 這是刻意的：這套系統要放在公網，而固定的預設密碼（`admin/admin` 之類）是全世界被掃描機器人打得最兇的入口——你自己寫報告時，這種發現大概都是列 Critical 的。
> 想自己指定密碼：`ADMIN_PASSWORD=你的密碼`（此時不會強制更改，因為那是你自己的選擇）。

管理員按側邊欄「👥 帳號管理」可以：

- **看到所有帳號**：權限、啟用狀態、筆記／資料夾／圖片數量、分享出／收到的數量、最後登入時間
- **停用 / 啟用帳號**：停用會**立刻踢掉對方所有登入中的 session**，且無法再登入
- **調整權限**：設為管理員 / 取消管理員
- **刪除帳號**：連同該帳號的所有筆記、資料夾、圖片一併永久刪除

> [!IMPORTANT]
> **管理員看不到任何人的筆記內容**，只看得到數量統計。要讀同事的報告，仍然得請對方分享給你。伺服器端根本沒有提供「讀取他人筆記」的 API。

防呆機制（避免把系統鎖死）：不能停用／降級／刪除**自己**；不能停用或降級**最後一個管理員**。

一般使用者按 🔑 可以自行更改密碼。

## 帳號與分享

- **登入 / 登出**：右上角顯示目前帳號，⏻ 登出。
- **隔離**：每個帳號只看得到自己的筆記與資料夾。
- **分享**：在自己的筆記上按右鍵 →「👥 分享…」→ 輸入對方帳號、選 **唯讀** 或 **可編輯**。
- 對方會在側邊欄的「**分享給我的**」看到，唯讀者的編輯器會鎖住並顯示橫幅。
- **可編輯者**能改內容，但**不能刪除**筆記，也不能把它搬進自己的資料夾——刪除永遠只有擁有者能做。
- 擁有者可隨時撤銷；收到分享的人也可以自行「移除這個分享」。
- 資料夾**不會**被分享，只有單篇筆記會。

## 功能

1. **多人帳號與分享** — 登入後只看得到自己的東西，可把單篇筆記分享給其他帳號，見上方〈帳號與分享〉。
2. **筆記管理** — 左側樹狀清單，可建立筆記與（巢狀）資料夾；右鍵可重新命名 / 刪除 / 複製；拖曳可搬移筆記或資料夾。
3. **全文搜尋** — 側邊欄搜尋框（`Ctrl/Cmd + K`）同時搜尋**標題與內文**，見下方〈全文搜尋〉。
4. **三種檢視** — 分割（左編輯右預覽）／純編輯／純預覽，即時渲染。
5. **GitHub 相容語法** — GFM（表格、任務清單、依語言高亮的程式碼區塊）＋ Callout（飽和色塊、圓角）：
   ```
   > [!NOTE]
   > 提示內容。也支援 [!TIP] [!IMPORTANT] [!WARNING] [!CAUTION]
   ```
6. **互動式表格（類 Notion）** — 工具列的 ▦ 會跳出**格子選單**，滑過選 N 欄 × M 列即可插入。游標移進表格時會浮出表格工具列：
   - 上/下插入列、左/右插入欄、刪除列 / 欄 / 整個表格
   - 設定該欄靠左 / 置中 / 靠右
   - **✨ 整理對齊**：自動把欄寬對齊（支援中文全形字寬），排版整齊
7. **程式碼區塊行號** — CodiMD 語法，在語言後加 `=`：
   ```
   ```js=      ← 顯示行號
   ```js=10    ← 從第 10 行開始
   ```
   也可用 `/codeln` 片段快速插入。
8. **語法自動完成** —
   - 輸入 `/` 叫出片段選單（`/note`、`/risk`、`/code`、`/codeln`、`/table`、`/todo`、`/wiki` …）
   - 輸入 `[[` 叫出筆記選單，連結到其他筆記
   - 程式碼圍籬輸入 ` ```py ` 自動補全語言名稱
   - 在引言行輸入 `>` 會跳出 callout / 弱點等級選單（用 **Tab** 或滑鼠選取；按 Enter 則是正常換行）
   - 自動配對括號 / 引號、清單與引言自動接續、`Tab` 縮排
9. **筆記互相連結** — 用 `[[筆記標題]]` 互相連結，見下方〈筆記連結〉。
10. **列印預覽 / 輸出 PDF** — 點工具列「⭳ PDF」會開啟**列印預覽**，可切換樣式主題與浮水印，見下方〈列印預覽與 PDF〉。
11. **插入圖片** — 兩種方式，系統都會自動存檔並顯示：
   - 在編輯器 `Ctrl/Cmd+V` 貼上
   - 直接把圖片檔案**拖曳**進編輯器
12. **截圖標註** — 在圖片上畫箭頭 / 方框、加文字、用馬賽克或塗黑遮住敏感資料，見下方〈截圖標註〉。
13. **弱點標記與總覽表** — 用 `> [!RISK:HIGH]` 標記弱點，PDF 自動產生依嚴重度排序的總覽表，見下方〈弱點標記與總覽表〉。

## 弱點標記與總覽表

用 `> [!RISK:等級]` 標記弱點，後面接弱點名稱：

```
> [!RISK:CRITICAL] EternalBlue RCE (MS17-010)
> DC01 未修補 SMBv1，可直接取得 SYSTEM。
```

等級有 **CRITICAL / HIGH / MEDIUM / LOW / INFO**（在引言行輸入 `>` 或 `[!` 會跳出選單，也可用 `/risk`）。

- 畫面上會渲染成帶**等級徽章**與**自動編號**（F-01、F-02…）的弱點區塊。
- 輸出 PDF 時，會在目錄後自動產生一頁 **Findings Summary**：
  - 依嚴重度排序（Critical → Info），列出編號、等級、名稱與**該弱點所在的頁碼**
  - 上方顯示各等級的數量統計
- 一般的 `[!NOTE]` 等 callout **不會**被算成弱點。

## 列印預覽與 PDF

點工具列「⭳ PDF」不再直接跳出列印視窗，而是先開啟**列印預覽**——裡面顯示的就是實際會印出來的分頁結果（封面、目錄含頁碼、頁碼與頁首），並即時顯示總頁數。調整滿意後再按「⭳ 列印 / 另存 PDF」，在列印視窗選「另存為 PDF」。

### 樣式主題（11 種）

| 主題 | 封面 |
| --- | --- |
| **科技風（淺色）** | 淺灰底＋左側藍色色條，與軟體介面同一套視覺（預設） |
| **經典藍（漸層）** | 藍色漸層滿版，白色標題 |
| **極簡黑白** | 白底黑字，上下粗線 |
| **深色封面** | 近黑底、亮藍點綴 |
| **紅隊（暗紅）** | 暗紅漸層＋紅色色條，紅隊報告用 |
| **企業襯線（白底）** | 白底襯線標題＋底部深藍色帶，顧問公司風格 |
| **終端機（等寬綠字）** | 黑底綠色等寬字，帶掃描線與 `>` 提示符 |
| **青綠漸層** | 青綠到深藍綠漸層 |
| **藍圖格線** | 深藍底＋工程圖格線 |
| **框線（正式）** | 白底雙線外框、置中襯線標題 |
| **琥珀（暖色）** | 米色底＋琥珀色頂帶 |

主題只改**封面與強調色**，內文排版不受影響——同一篇筆記換主題不會重新斷頁跑版（各主題頁數一致）。

### 浮水印

可選 **DRAFT**、**CONFIDENTIAL**、**INTERNAL USE ONLY**，或自訂文字（例如客戶名稱）。浮水印會以斜向淡色印在**每一頁內容上，但不會蓋住封面**。

### 封面欄位

可填 **客戶名稱、版本、作者**，會印在封面（連同 Generated / Last Updated 一起對齊排列）。這些欄位跟著**該篇筆記**走，不是全域設定。

### TLP 分類標記

支援 **TLP:CLEAR / GREEN / AMBER / AMBER+STRICT / RED**（FIRST TLP 2.0）。選定後：

- **封面**右上角出現官方樣式的標記（黑底＋等級色）
- **每一頁頁首**左上角印出該標記

### 其他

- **H1 章節從新頁開始** — 勾選後每個 `#` 大章節都會另起一頁（第一個章節不會多出空白頁）。
- 主題、浮水印、H1 換頁為全域設定；封面欄位與 TLP 跟著筆記走。設定都會記住。

## 截圖標註

在預覽區把滑鼠移到任一張圖片上，右上角會出現 **✎ 標註**，點下去就能直接在圖上編輯，不必離開這個軟體：

| 工具 | 用途 |
| --- | --- |
| ↗ 箭頭 | 指出重點位置 |
| ▭ 方框 | 框住某一段輸出 |
| T 文字 | 加說明文字（自動加白邊，任何底色都看得清楚） |
| ▩ 馬賽克 | 把密碼、雜湊、真實 IP 打上馬賽克 |
| ■ 塗黑 | 直接塗黑遮蔽 |

- 8 種顏色、3 種線寬；`Ctrl/Cmd + Z` 復原、`Ctrl/Cmd + Shift + Z` 重做、`Esc` 取消。
- 拖曳畫出圖形；文字工具是**點一下**再輸入，按 Enter 完成。
- **標註可以再編輯**——原圖會完整保留，標註以向量形式存起來，下次打開還能改或刪掉單一標註，也可以按「還原成原圖」整個還原。
- 標註後的圖片會直接反映在筆記與 **PDF 輸出**中；標註按鈕本身不會出現在 PDF 裡。

> [!NOTE]
> 遮蔽（馬賽克 / 塗黑）會實際烙印在輸出的圖片上，所以匯出的 PDF 看不到被遮的內容。但為了讓標註可以反悔，**原圖仍保留在本機資料庫中**。

## 全文搜尋

側邊欄的搜尋框可同時搜尋**筆記標題與內文**（按 `Ctrl/Cmd + K` 可從任何地方跳到搜尋框）。

- **即時結果** — 一邊打字一邊出現結果，並顯示命中處的**內文摘要**與**高亮**，不必先開啟筆記就知道命中在哪。
- **多關鍵字** — 用空白分隔（例如 `nmap ad`），只列出**全部關鍵字都命中**的筆記。
- **排序** — 標題命中優先於內文命中，其次依命中次數與最近編輯時間。
- **鍵盤操作** — `↑` `↓` 選擇（會循環）、`Enter` 開啟、`Esc` 清除並回到編輯器。
- 搜尋不區分大小寫，中文可直接搜尋；`$1.50`、`(a|b)*` 等特殊字元會**照字面比對**，不會被當成正規表示式。
- 剛打字**還沒自動儲存**的內容也搜得到。

## 筆記連結

用 `[[筆記標題]]` 連到另一篇筆記：

```
詳見 [[Active Directory 列舉]]
用別名顯示：[[Active Directory 列舉|AD 列舉技巧]]
```

- 輸入 `[[` 會跳出**筆記選單**，用 Tab／Enter 或滑鼠選取即可補完（也可用工具列「⧉ 筆記」或 `/wiki`）。
- 預覽區點擊連結即可跳到該篇筆記。
- 連結到**還不存在**的筆記時會顯示紅色虛線，點一下就直接建立那篇筆記——適合先列大綱、之後再逐篇補寫。
- 每篇筆記的預覽下方會列出**反向連結**（哪些筆記連到這一篇）。
- 在左側樹狀清單重新命名筆記時，其他筆記中指向它的連結會**自動跟著更新**。

## 報告範本產生器

側邊欄「報告範本產生器」會跳出表單，可選擇考試 / 實驗室範本：

| 範本 | 機構 |
| --- | --- |
| PEN-200（OSCP） | OffSec |
| PEN-300（OSEP） | OffSec |
| CPENT / LPT (Master) | EC-Council |
| VHL | Virtual Hacking Labs |

> [!NOTE]
> CPENT 與 LPT (Master) 共用同一份範本——同一場考試，達到及格分數得到 CPENT，**90% 以上**則加發 LPT (Master)，報告結構相同。

表單會依所選範本詢問對應欄位：
- 考生 / 學員編號、姓名、日期、封面 Logo（選填）
- 靶機 IP（可增減）／對外主機／實驗室機器／目標應用（視範本而定）
- AD 網域名稱與 AD Set 主機（主機名 + IP，可增減）

送出後自動生成完整的報告骨架，包含：
- 罐頭訊息（Introduction / Objective / Requirements）
- High-Level Summary、Scope、Methodologies
- 每台靶機的列舉表格、Initial Access、local.txt、提權、proof.txt 區塊
- Active Directory Set 的主機列舉與逐台攻擊段落
- Appendix — Proof Summary

### CPENT：依五個 Range 分章

CPENT（現為 CPENT AI）的實作靶場分成五個 Range，因此這份範本**不採用 OSCP 那種「靶機清單」結構**，而是依 Range 分章，並在 Scope 產生一張總表列出每個 Range 的目標數：

| # | Range | 章節內容 |
| --- | --- | --- |
| 1 | **Active Directory** | 主機列舉表、逐台攻擊、憑證收集、Domain Compromise |
| 2 | **Binaries** | 漏洞分析、Crash / Root Cause、Exploit 開發、Proof |
| 3 | **IoT** | 裝置探測、取得存取、**韌體抽取**（binwalk）、逆向、協定分析 |
| 4 | **Web** | 概述、弱點、重現步驟、PoC、Proof |
| 5 | **CTF** | 列舉表、Exploitation、提權、Flag |

表單會分別詢問這五個 Range 的目標（AD 主機、二進位、IoT 裝置、Web 應用、CTF 主機），沒填的 Range 仍會保留章節骨架。

接著用工具列「⭳ PDF」即可輸出精美、含封面 / 目錄 / 頁碼的報告。

## 介面風格

## 技術

**前端**：無需建置，相依函式庫已放於 `vendor/`（marked、DOMPurify、highlight.js、paged.js）。

**後端**：`server/`，**零 npm 依賴**——只用 Node 內建的 `node:http`、`node:sqlite`、`node:crypto`。

```
server/
  server.js   HTTP、路由、靜態檔案、安全標頭
  api.js      筆記／資料夾／圖片／分享 ＋ 權限檢查
  auth.js     scrypt 密碼、session、限速
  db.js       schema 與查詢（全部 prepared statement）
  config.js   環境變數設定
  data/       data.db（請備份）
```
